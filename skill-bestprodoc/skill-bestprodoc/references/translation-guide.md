## Sentence Patterns

### Overview

**Chinese**: 本文档基于[项目名称]整理，面向[目标受众]，重点展示[核心能力]。
**English**: This document is based on [Project Name], oriented towards [Target Audience], focusing on demonstrating [Core Capability].

**Chinese**: [项目名称]是[场景描述]的典型案例。
**English**: [Project Name] is a typical case of [Scenario Description].

### Challenges

**Chinese**: [系统/场景]往往面临以下挑战：
**English**: [System/Scenario] often faces the following challenges:

**Chinese**: [挑战描述]对[业务/系统]提出了高要求。
**English**: [Challenge Description] places high requirements on [Business/System].

### Solutions

**Chinese**: HyperBDR通过[能力/功能]提供解决方案：
**English**: HyperBDR provides solutions through [Capability/Function]:

**Chinese**: 采用[方案/方法]，实现[目标/效果]。
**English**: Adopt [Solution/Method] to achieve [Goal/Effect].

### Implementation

**Chinese**: 在[阶段]阶段，本项目采用[策略/方法]：
**English**: In the [Phase] phase, this project adopts [Strategy/Method]:

**Chinese**: [步骤名称]：[时间] | [关键动作] | [目的]
**English**: [Step Name]: [Time] | [Key Action] | [Purpose]

### Results

**Chinese**: 采用[方案]，在[场景]过程中可达到以下效果：
**English**: Adopting [Solution], the following results can be achieved during [Scenario]:

**Chinese**: [指标]：[结果] | [HyperBDR贡献]
**English**: [Metric]: [Result] | [HyperBDR Contribution]

### Summary

**Chinese**: 本项目成功验证了[能力]在[场景]下的有效性。
**English**: This project successfully verified the effectiveness of [Capability] in [Scenario].

**Chinese**: 项目取得的关键成果如下：
**English**: The key achievements of the project are as follows:

## Translation Best Practices

### 1. Maintain Technical Accuracy
- Use standard technical terminology
- Preserve technical specifications
- Keep measurements and metrics unchanged
- Maintain technical precision

### 2. Ensure Readability
- Use natural English phrasing
- Avoid literal word-for-word translation
- Consider context and audience
- Use appropriate technical writing style

### 3. Preserve Structure
- Maintain document hierarchy
- Keep section organization
- Preserve table formats
- Maintain list structures

### 4. Handle Special Cases
- **Proper Names**: Keep original names (e.g., VMware, SAP HANA)
- **Acronyms**: Keep standard acronyms (e.g., RPO, RTO, DR)
- **Metrics**: Keep metric values and units unchanged
- **File Paths**: Keep file paths unchanged

### 5. Review and Validate
- Check for translation consistency
- Verify technical accuracy
- Ensure grammatical correctness
- Validate formatting preservation

## Common Translation Pitfalls

### Avoid These Mistakes

1. **Literal Translation**
   - ❌ "Cloud One Click" instead of "Boot in Cloud"
   - ✅ Use established terminology

2. **Inconsistent Terminology**
   - ❌ Using "disaster recovery" and "DR" interchangeably
   - ✅ Establish consistent terminology usage

3. **Loss of Technical Precision**
   - ❌ "about 2 hours" instead of "< 2 hours"
   - ✅ Maintain precision in metrics and specifications

4. **Cultural Differences**
   - ❌ Direct translation of idioms or cultural references
   - ✅ Adapt to target audience context

5. **Formatting Issues**
   - ❌ Losing table structure or list formatting
   - ✅ Preserve document structure
